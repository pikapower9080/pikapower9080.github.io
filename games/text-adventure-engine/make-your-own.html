Make your own text adventure!

  1. First you'll need to learn about JSON syntax. I know this looks like code, but I can assure you that it's not.
    • Every JSON document starts with two opening and closing brackets.
    • You start by defining a key in quotes. Then put a colon after it, then a space and it's value.
    • Your key and value could be anything, but if you want to use quotes in the value you have to put a backslash
    • Values don't have to be strings, they can also be booleans (true or false), or arrays of other values.
    • You can also store sub json tables inside of values and arrays.
    • That's all you need to know! Congrats, you're a json master!
  2. Next, check out the example story.
  3. Reading this should help you understand how this works. I'd highly recommend you use this as your base, and simply modify the values
  4. This example uses every option and attribute possible, so you should know about everything.
  5. You can test your story by pasting it into the box on the main page. Please do not type your document in that box! It won't save! Use something like visual studio code and create a file on your computer ending in .json

Sharing

There are two ways you can share your story with your friends and the world. The first way is to send them the json data using your favorite messaging app along with the url and explain how to use the data. This isn't a very practical method, but it's good for testing. The better way is to sign up for a github account to create a gist. Once you have that set up, go to gist.new. Set the gist description to anything you want and set the first file's name to game.json. Paste your json data into the box.

Important: Before you publish, click the dropdown arrow next to create secret gist and select create public gist. You'll be taken to a page with a url that looks something like "gist.github.com/username/(random letters & numbers)". Copy those random letters and numbers and press the load gist button. That's it! You can now share the page's url and your friends will automatically be loaded into your story.

Tips

Locked Doors

Create a locked door by using the required argument. The argument takes an array of item ids that are needed to open the door. You will also need to set requiredText to the text that will show when you can't unlock the door. New items will not be able to be picked up without the required items but you will still be able to use adjacent rooms so that the player can go back. If you want an adjacent room to only be available to players who have unlocked the door, use the requiredAdjacent attribute.

You can use this method for more than just doors too, in the example story you will find a closet that needs a broom to clear out the cobwebs.

Intro (provide context)

You can use the intro setting to provide context to your players. Create an array in the top level of your story document called intro. Each element in the array is a new message, and the player will click to move between them. Once all of the intro messages have been shown, the game starts.

Inventory Space

You can set options.inventorySlots to the max number of items you want your players to have. For example, if you set inventorySlots to 5, once the player had 5 items they wouldn't be able to pick up any more. Setting the canDrop attribute to true on an item will allow the player to drop it, freeing up one inventory space. Add the addInventorySlots attribute to an item to have it grant that number of inventory slots to the player when it's picked up. Items with this attribute cannot be dropped, regardless of the canDrop attribute.

Item tooltips

You can set the tooltip attribute of an item to add an optional description of the item. It will show when you're about to pick it up and when you hover over it in your inventory.
